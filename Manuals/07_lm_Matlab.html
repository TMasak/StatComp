Now it is time to fit the linear model y ~ race + fire + log(income).
While we would have done this in R and Python by using the symbolic formula above, this formula captures the nature for the expected value of y (conditionally on the covariates). Linear models are about modeling the conditional mean of the response as a linear function of the covariates, hence a specific model is given by a formula for the conditional mean.
An equivalent (linear algebra) point of view is to provide the model matrix X. Since Matlab is all about linear algebra and matrices are its main objects, it is simpler in Matlab to specify a linear model by specifying the model matrix (and the response, of course).
y = chredlin(:,6); % choose `involact` from the table above as the response
y = table2array(y); % transform table into an array (vector)
X = chredlin(:,[2,3,7]); % choose `race`, `fire` and `income` from the table above as the covariates
X = table2array(X); % transform table into an array (matrix)
X(:,3) = log(X(:,3)); % log-transform `income`
lm2 = fitlm(X,y) % fit the linear model

Interestingly, race is no longer significant!
There is no particular reason why this model should be preferred over the models we saw in R or Python, so we should not just conclude that insurance companies base their decisions on sound reasons (such as number of fires in the area). Also note that the p-value is still quite small. But at the same time we also should not just conclude the opposite based on the models in R or Python. Obviously, the conclusions depend on a couple of observations and some modeling choices, so at this point we should go back to data exploration (and take a closer look at the influential observations) and be self-critical about the choices we make as modellers.
